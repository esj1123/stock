# ACCEPTANCE_TRACE.md

## Purpose
06_Stock 프로그램 개발 계약과 검증 증거를 1:1로 연결하는 개발용 trace 문서다.
문서 운영 체계를 늘리기 위한 파일이 아니라, Codex가 import pipeline을 수정할 때 깨면 안 되는 데이터 계약과 safety gate를 고정한다.

## Trace Fields
- `Trace ID`: 고정 식별자
- `Category`: `data_contract` | `safety_contract` | `quality_gate` | `repo_hygiene`
- `Requirement`: 반드시 유지해야 하는 개발 계약
- `Evidence`: 테스트, quality gate, 산출물, 또는 파일 위치
- `Status`: PASS | WARN | FAIL | PENDING
- `Last checked`: 마지막 확인일
- `Notes`: 보충 설명
