# Codex_03_Triage_Unclassified

목적: 임포트 후 `UNCLASSIFIED`(분류 실패) 항목을 **버리지 않고**, 어떤 유형이 섞여있는지 정리해 “다음 튜닝”을 쉽게 만듭니다.

> 이 카드는 **분석/정리만** 합니다. (코드 자동 수정 X)

---

## Copypaste Prompt (Codex에 그대로 붙여넣기)

```text
[목표]
- 70_Imports/review/ 및 (있다면) 70_Imports/processed/ 의 unclassified 관련 파일을 읽고,
  "무슨 종류의 행이 분류 실패했는지"를 사람이 판단하기 쉽게 요약하세요.

[사전 점검]
1) 70_Imports/review/ 폴더가 있는지 확인하세요. 없으면 "분류 실패 없음(또는 미생성)"으로 보고하고 종료.
2) review 폴더에 파일이 0개면 "분류 실패 없음"으로 보고하고 종료.

[작업]
1) review 폴더의 파일들을 열어, 행/레코드에서 다음을 최대한 추출하세요(가능한 범위에서만):
   - 거래내용/적요/구분/비고/메모 계열 텍스트
   - 금액/수량/통화(있으면)
   - 날짜
2) 분류 실패를 '키워드 기반'으로 묶어서 그룹을 만드세요.
   예: 환전/외화매수, 이자, 수수료/세금, 대체입출고, 기타
   (정확히 모르면 '기타'로)

[산출물]
- 아래 파일을 새로 생성하세요(없으면 폴더도 생성):
  - 70_Imports/review/Unclassified_Triage_YYYY-MM-DD.md

- 노트에 포함할 섹션(필수):
  1) 파일 목록(읽은 review 파일들)
  2) 그룹 요약(그룹명 / 대략 개수 / 대표 키워드 3~10개)
  3) "임포터에 추가하면 좋을 규칙" 제안(코드 변경은 하지 말고, 규칙 아이디어만)
  4) (가능하면) COLUMN_SYNONYMS에 추가하면 좋을 열 이름 후보(추정 금지, 실제 파일에 있는 헤더만)

[주의]
- 코드 파일(70_Imports/scripts/namoo_excel_import.py) 수정 금지.
- raw 엑셀 수정 금지.

끝나면, 생성한 triage 노트 경로를 마지막 줄에 알려주세요.
```
